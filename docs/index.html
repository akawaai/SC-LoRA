Steering-Conditioned LoRA for Continual Multi-Domain Fine-Tuning

SC-LoRA routes low-rank adaptation through sparse autoencoder features and steering prototypes, enabling domain specialization while limiting catastrophic forgetting on prior tasks.

Abstract

Parameter-efficient fine-tuning methods such as LoRA adapt a shared low-rank update across all inputs, which makes them efficient but poorly suited to continual learning across heterogeneous domains. Steering-Conditioned LoRA (SC-LoRA) conditions adapter selection on interpretable latent structure: layerwise sparse autoencoder features, a taxonomy of general versus domain-specific units, and steering prototypes that encode where each domain lives in feature space. A bank of specialist LoRA adapters is routed, ranked, and compacted dynamically, with optional replay to preserve prior domains. The result is a modular PEFT stack that separates what to adapt from when and how much to adapt.

Why SC-LoRA

SAE-Guided Routing

Adapter gates read sparse feature activations instead of raw hidden states, grounding routing in mechanistically meaningful structure rather than opaque logits alone.

Dynamic Rank Control

Rank controllers allocate capacity per specialist and per layer, expanding where a domain needs expressivity and compressing elsewhere to stay within a parameter budget.

Continual Learning Ready

Preservation losses, replay buffers, and memory policies for spawning, merging, and pruning specialists target reduced interference when new domains arrive sequentially.

Method

SC-LoRA treats continual fine-tuning as conditional low-rank adaptation over a frozen backbone. The pipeline below is implemented in the open-source sc_lora package.

1. Activations Hook selected layers on a frozen LLM
→
2. SAE Train layerwise sparse autoencoders
→
3. Taxonomy Split general vs domain features
4. Prototypes Build steering vectors per domain
→
5. SC-LoRA Bank Route specialists with gates & rank
→
6. Evaluation Forgetting, routing, mechanistic metrics
  1. Collect activations from probe, general, and domain datasets via forward hooks.
  2. Train sparse autoencoders per hooked layer to obtain feature codes z.
  3. Estimate general and domain feature masks; build steering prototypes and covariances.
  4. Initialize a conditional LoRA adapter bank with gate, rank, and budget controllers.
  5. Train sequentially with task, preservation, steering, and sparsity regularizers; optionally replay prior domains.
  6. Evaluate routing specialization, sequential forgetting, interference, and SAE alignment.

Conditional update

Each forward pass selects specialists and effective rank from SAE features and prototypes:

ΔW = Σ_k g_k(z, p_d) · B_k A_k rank_k = f_rank(z, budget)

Gates g_k depend on sparse codes; prototypes p_d steer domain context; memory policy spawns, merges, or prunes specialists over training.

Core Library

The repository ships a modular PyTorch implementation designed to be composed with your own models and dataloaders.

Contributions

Steering-conditioned adaptation

Connects PEFT to sparse feature geometry via SAE codes and domain prototypes, making adapter routing interpretable and domain-aware.

Modular continual-learning stack

Combines specialist banks, dynamic rank, memory compaction, and replay in one library suitable for sequential domain fine-tuning experiments.
